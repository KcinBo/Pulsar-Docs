## Verification checklist

Before interacting with any future address:

1. Open this page from the official Pulsar documentation domain.
2. Confirm the network.
3. Compare the full address—not only its first and last characters.
4. Open the verified block-explorer page.
5. Confirm that the application, documentation, and explorer all match.
6. Use a small test transaction when practical.

Official deployments should be added only after verification, audit status should be linked beside each contract, and proxy or upgrade authority should be disclosed.

> Anyone can create a token named PULSAR or QUASAR. A name, symbol, logo, or token-list entry does not prove authenticity.
